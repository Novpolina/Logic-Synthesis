## Структура проекта
* `pwm_generator.v` — параметризуемый 8-битный ШИМ-генератор.
* `comparator.v` — 2-битный синхронный компаратор.
* `cg_bypass.v` — заглушка для обхода несмапленных защелок в модуле Clock Gating процессора.
* `synth_scr1.tcl` / `timing_scr1.tcl` — автоматизированные скрипты синтеза и STA для ядра SCR1.
* Сгенерированные нетлисты (`netlist_out_pwm.v`, `netlist_scr1.v`) и PDF-отчет.

## Основные результаты
**1. PWM Generator (8-bit):**
* Успешный маппинг на 46 ячеек.
* Временные ограничения (100 MHz) успешно выполнены (Slack: +9.02 ns).

**2. 32-bit RISC-V Core (SCR1):**
* Исходный SystemVerilog код успешно конвертирован (через `sv2v`) и синтезирован.
* Применен метод ручного маппинга для корректного синтеза модуля Clock Gating.
* **Тайминги (Gated Clock):** Успешно выполнены (Slack: +9.73 ns).
* **Тайминги (Критический путь):** Выявлен путь (IFU -> IDU -> IALU) с задержкой 32.26 ns. Базовая расчетная частота без физической оптимизации на техпроцессе 130нм составляет **~31 МГц**.

## Запуск сложных дизайнов (RISC-V SCR1)
Для синтеза сложных архитектур на SystemVerilog используется утилита `sv2v` для предварительного сплющивания кода:

```bash
# 1. SystemVerilog в Verilog-2005
./sv2v-Linux/sv2v -I./scr1/src/includes $(find ./scr1/src/core ./scr1/src/top -type f -name "*.sv") > scr1_flattened.v

# 2. Комплексный синтез процессора (с заменой Clock Gate)
yosys -s synth_scr1.tcl

# 3. Анализ критического пути
sta timing_scr1.tcl
```
